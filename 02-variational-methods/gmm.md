# Gaussian Mixture Models

## Abbreviation

**GMM = Gaussian Mixture Model**

## One-Line Definition

A Gaussian Mixture Model assumes data is generated from a mixture of multiple Gaussian distributions.

## Core Idea

Instead of assuming all data comes from one Gaussian, a GMM assumes there are several Gaussian components.

Each data point is generated by:

1. Choosing a component
2. Sampling from that component's Gaussian distribution

## Mathematical Form

```math
p(x) = \sum_{k=1}^{K} \pi_k \mathcal{N}(x \mid \mu_k, \Sigma_k)
```

where:

- `K` = number of mixture components
- `pi_k` = mixture weight for component `k`
- `mu_k` = mean of component `k`
- `Sigma_k` = covariance matrix of component `k`
- `N(x | mu_k, Sigma_k)` = Gaussian density

## Latent Variable

The hidden variable is the component assignment:

```math
z \in \{1, 2, \dots, K\}
```

## Training

GMMs are commonly trained using the EM algorithm.

## EM Algorithm

**EM = Expectation-Maximization**

### E-Step

Estimate the probability that each point belongs to each component.

```math
\gamma_{ik} = p(z_i = k \mid x_i)
```

### M-Step

Update the Gaussian parameters using the estimated assignments.

Parameters updated:

- Mixture weights
- Means
- Covariances

## Strengths

- Interpretable
- Useful for clustering
- Good introduction to latent variable modeling
- Has a clear probabilistic foundation

## Weaknesses

- Assumes Gaussian components
- Does not scale well to very high-dimensional complex data
- Limited expressive power compared with neural generative models

## Why It Matters for Generative AI

GMMs are not frontier generative AI models, but they teach the core idea of hidden structure behind observed data.
